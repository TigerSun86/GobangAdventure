# BR3 玩法术语表

## 文档目的

本文档是 BR3 玩法设计讨论中的术语参考文档。

本文记录：

- 玩法讨论中使用的中文正式术语；
- Chat D / Chat A 讨论中可使用的中文简称；
- 对应英文术语，原则上用于指导未来代码命名；
- 每个术语当前是已接受术语，还是仍处于候选状态。

本文档的目标是在 Chat A 审阅后，成为 BR3 玩法术语的统一 source of truth。若其他玩法文档中存在重复术语定义，后续应删除重复定义，或改为引用本文档。

---

## 状态说明

| 状态 | 含义 |
|---|---|
| 已接受 | 当前规则、文档或长期讨论中已经采用的术语。 |
| 候选 | 当前设计讨论中有用，但尚未锁定进入正式规则或实现的术语。后续可由 Chat A 审阅后删除、调整或转为已接受。 |

---

## 使用原则

1. 玩法设计文档中优先使用中文正式术语。
2. 简称可用于聊天、表格、草稿、头脑风暴和空间受限的说明。
3. 英文术语原则上指导未来代码命名，除非实现时存在明确理由不采用。
4. 候选术语可以用于 Chat D / Chat A 讨论，但不代表相关系统已经进入正式规则。
5. 卡牌上的能力在玩法讨论中称为 **特质 / Trait**。即使从抽象角度看特质也可以理解为某种修正，玩法讨论中仍保留“特质”这一独立术语。
6. **修正 / Modifier** 是规则修改的上位概念；**全局修正 / Global Modifier** 是其中一种。
7. 描述不同层级的结果时，应明确区分：

   * 槽位层面的槽位胜利、槽位失败与槽位平局；
   * 轮末结果判定中的玩家死亡与击败敌人；
   * 一局层面的一局胜利与一局失败。
     不应只使用“胜利”“失败”或“死亡”而不说明其作用层级。

# 1. 构筑与修正术语

# 4. 结果与终局术语

本节区分：

1. 单个槽位的结果；
2. 一轮完整结算后的角色与敌人状态判定；
3. 一局游戏的最终结果。

这些概念处于不同层级，不应混用。

## 4.1 槽位结果

| 状态  | 中文正式术语 | 简称 | 英文术语         | 说明                                            |
| --- | ------ | -- | ------------ | --------------------------------------------- |
| 已接受 | 槽位结果   | 槽果 | Slot Outcome | 某个槽位在一次槽位结算中的结果。简称“槽果”可用，但不推荐频繁使用，除非空间很有限。    |
| 已接受 | 槽位胜利   | 槽胜 | Slot Win     | 玩家牌在该槽位克制敌方牌。描述槽位层面的胜负时，应使用“槽位胜利”，避免单独说“获胜”。  |
| 已接受 | 槽位失败   | 槽败 | Slot Loss    | 玩家牌在该槽位被敌方牌克制。描述槽位层面的失败时，应使用“槽位失败”，避免和一局失败混淆。 |
| 已接受 | 槽位平局   | 槽平 | Slot Tie     | 玩家牌与敌方牌在该槽位的基础类型相同。                           |

## 4.2 轮末结果判定

| 状态  | 中文正式术语 | 简称   | 英文术语                       | 说明                                                                                         |
| --- | ------ | ---- | -------------------------- | ------------------------------------------------------------------------------------------ |
| 已接受 | 轮末结果判定 | 轮末判定 | End-of-Round Outcome Check | 当前七个轮内结算阶段全部完成，并且本轮伤害、治疗与结算后效果全部应用后进行的结果判定。依次检查玩家死亡、敌人被击败以及 battle 是否继续或结束。该判定不属于新的轮内结算阶段。 |
| 已接受 | 玩家死亡   | 玩家死亡 | Player Death               | 一轮完整结算后，玩家最终 HP 为 0 或以下。玩家 HP 在合并伤害后暂时降至 0 或以下，但被本轮结算后治疗恢复至 0 以上时，不视为玩家死亡。玩家死亡会直接导致一局失败。   |
| 已接受 | 击败敌人   | 击败   | Enemy Defeated             | 一轮完整结算后，玩家最终 HP 大于 0，并且当前敌人最终 HP 为 0 或以下。若玩家与敌人同时降至 0 或以下，则不视为一次成功击败敌人。                    |

## 4.3 一局结果

| 状态  | 中文正式术语 | 简称 | 英文术语        | 说明                                                                                                |
| --- | ------ | -- | ----------- | ------------------------------------------------------------------------------------------------- |
| 已接受 | 一局胜利   | 局胜 | Run Victory | 玩家存活并击败当前一局中的最终敌人后，一局游戏成功结束。                                                                      |
| 已接受 | 一局失败   | 局败 | Run Defeat  | 一局游戏失败并进入终止状态。当前正式原因包括：玩家在一轮完整结算后死亡；或玩家存活，但未能在当前敌人的 battle 上限内击败该敌人。若玩家与敌人同时降至 0 或以下，玩家死亡优先，本局失败。 |

| 候选 | 治疗源 | 治源 | Healing Source | 一段治疗归属的来源。建议来源值包括牌、物品、角色、敌人、系统。 |

# 9. 修正描述字段术语

这些术语主要用于描述、比较和评估修正。它们适用于 Chat D 头脑风暴、Chat A 筛选评估，以及未来可能的实现规格讨论。

| 状态 | 中文正式术语 | 简称 | 英文术语 | 说明 |
|---|---|---|---|---|
| 候选 | 触发条件 | 触发 | Trigger Condition | 修正开始检查或触发的条件。 |
| 候选 | 作用时机 | 时机 | Timing | 修正在什么时候结算，例如槽位结果产生后、本轮结束时、战斗结束时、奖励生成时。 |
| 候选 | 作用对象类型 | 对象类型 | Target Kind | 被影响对象的大类，例如玩家、敌人、牌、槽位、奖励选项、奖励生成规则。 |
| 候选 | 作用范围 | 范围 | Target Scope | 在哪里或以什么范围选择作用对象，例如玩家侧、触发槽位、相邻槽位、当前敌人、下一次奖励报价。 |
| 候选 | 筛选条件 | 筛选 | Target Filter | 在作用范围内进一步筛选对象的条件，例如只影响 Upgrade 选项，或只影响拥有某个特质的牌。 |
| 候选 | 效果概述 | 概述 | Effect Summary | 对修正的一句话快速描述。应尽量包含触发条件、对象和主要效果。 |
| 候选 | 效果细则 | 细则 | Detailed Effect | 更精确的规则描述，用于消除歧义。 |
| 候选 | 持续时间 | 持续 | Duration | 效果持续多久，例如本次触发、本轮、下一轮、本场战斗、当前敌人阶段、一局游戏、永久。 |
| 候选 | 限制 | 限制 | Limit | 触发次数限制、是否叠加、每轮一次、每场战斗一次等额外限制。没有额外限制时可写“无”。 |
| 候选 | 设计意图 | 意图 | Design Intent | 该修正希望制造的玩家行为或体验。 |
| 候选 | 联动备注 | 联动 | Synergy Notes | 与其他规则、修正、特质可能产生的设计联动或涌现联动。 |
| 候选 | 相近修正区别 | 区别 | Distinction from Similar Modifiers | 说明该修正与容易混淆的相近修正有什么区别。 |
| 候选 | 风险 | 风险 | Risks | 平衡性、可读性、实现复杂度、UI 展示等风险。 |
